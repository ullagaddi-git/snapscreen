## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app in a broken or unrunnable state. After completing any phase, you should be able to run the app and see something working.
2. **Infrastructure before features.** Phase 0 sets up the complete project scaffold, tooling, and build pipeline so that every feature phase starts from a clean, working foundation.
3. **Magic moment first.** The core value — press a hotkey, record exactly the right monitor — is working by the end of Phase 1. Everything else builds on top of that.
4. **Test as you go.** Each task includes a verification step. Don't mark a task complete until you've manually verified the expected behavior.
5. **Progressive enhancement.** Start with the simplest working version, then layer on polish. An ugly working tray app beats a beautiful broken one.
6. **Review before proceeding.** After completing each phase, create a branch, commit, push, and open a PR. Use [CodeRabbit](https://coderabbit.ai) (free for public repos) to get automated code review before starting the next phase.
7. **Windows-first thinking.** Every interaction must feel native on Windows 11. When in doubt, use the Windows platform API or native HTML control rather than a custom implementation.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase. The prompt tells the agent exactly which docs to read and where to start.
2. **Read selectively:** Each phase lists its Reference sections — read only those parts of the PRD and vision doc. Don't load entire documents into context unnecessarily.
3. **Let it work:** The agent finds the first unchecked task, implements it, marks it `- [x]`, and moves to the next. One task at a time.
4. **One session = one phase (ideally):** Complete a full phase in one coding session for best continuity. If you stop mid-phase, the agent can resume from the last unchecked task.
5. **Push a PR for review:** When a phase is done, push the work as a PR and let [CodeRabbit](https://coderabbit.ai) (free for public repos) review it before starting the next phase. This catches issues before they compound.
6. **Need more context?** If a task references something not in the phase Reference sections, the agent should read just that specific section on demand.

### Session Tips

- **Verify each task before marking complete.** The verification step is part of the task, not optional.
- **Don't skip tasks.** Tasks within a phase are ordered to resolve dependencies. TASK-013 (recording start) must come before TASK-015 (global hotkey) because the hotkey calls the recorder.
- **Don't modify the PRD or vision doc from the coding agent.** Those are source of truth documents. If requirements need to change, update them manually and then brief the agent.
- **Track the status line.** After each phase, update: `**Status:** X/42 tasks complete` and `**Current Phase:** Phase N — Title`.

### Prompt Templates

**Starting a new phase:**
> "Read `docs/product-roadmap.md` and find the current phase. Read only the Reference sections listed for that phase from `docs/prd.md` and `docs/product-vision.md`. Start working on the first unchecked task. After completing each task, update the checkbox to `[x]` in the roadmap file. Continue through the phase."

**Resuming after a break:**
> "Read `docs/product-roadmap.md`. Find where we left off — the first unchecked task. Read only the Reference sections listed for the current phase from `docs/prd.md` and `docs/product-vision.md`. Continue from the first unchecked task."

**After completing a phase:**
> "Phase [N] is complete. Create a branch called `phase-{N}/{slug}`, commit all work, push, and open a PR targeting main. Title it 'Phase {N}: {Title}' and include the phase goal and completed task count in the body. List what to manually verify in the PR description."

**Fixing an issue:**
